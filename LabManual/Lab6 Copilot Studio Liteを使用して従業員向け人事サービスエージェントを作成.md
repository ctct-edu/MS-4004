# 演習：会社の従業員向けの人事セルフサービス エージェントを作成
---
あなたは、米国の複数の拠点に約 3,000 人の従業員を擁する中規模テクノロジ企業である Adatum Corporation の人事アナリストです。 Adatum の人事部門には、福利厚生、昇進、転勤、その他の会社ポリシーに関する問い合わせが毎月何百件も寄せられます。

コストを削減し、サービスを向上させるために、あなたは Microsoft 365 Copilot で人事セルフサービス エージェントを作成するように求められました。 このエージェントの目的は、公式の人事ポリシー ドキュメントをナレッジ ベースとして使用して、従業員の質問に答えることです。

> [!NOTE]
> この演習では、Copilot Studio Lite エクスペリエンスを使用して人事セルフサービス エージェントを作成します。 このシンプル エクスペリエンスは、日常業務を行うビジネス ユーザー向けに設計されており、プログラミング スキルは必要ありません。 一方で、より複雑で高度なエージェントを構築するソフトウェア開発者は、通常、Copilot Studio Full エクスペリエンスを使用します。

このタスクを完了するには、以下の手順を実行します。

1. Microsoft 365 のナビゲーション ウィンドウで **[新しいエージェント]** を選択します。 これにより、Copilot Studio の**エージェント ビルダー**が開き、**[新しいエージェント]** ページが表示されます。

2. **[新しいエージェント]** ページで、エージェントを作成するよう Copilot に指示します。 プロンプトでは、エージェントの名前と、エージェントの概要、対象ユーザー、それで実行したいことの一般的な説明を入力する必要があります。  
   このエージェントの場合、次のプロンプトを入力し、進む矢印 (送信) アイコンを選択してプロンプトを送信します。  

   ```prompt
   「HR Self-Service Assistant」というタイトルのエージェントを作成します。このエージェントの目的は、Adatum Corporation 従業員の人事に関連する質問に回答することです。エージェントは、ナレッジ ソースとしてこのエージェントに割り当てられた承認済みの人事ポリシー ドキュメントのみを使用する必要があります。
   ```

3. 進む矢印を選択してプロンプトを送信すると、新しいエージェントの**エージェント ビルダー** フォームが表示されます。 

   - 画面左側ではCopilot との会話を続けられます。 このタブは既定で表示されます。

   - 画面右側の **[構成]** タブでは、エージェントを動作させるための詳細な設定を定義できます。

   Copilot がエージェントを作成するまで 1 から 2 分待つと、エージェントの名前と説明が **[エージェント プレビュー]** ペインに表示されます。

4. フォームの上部にある **[構成]** タブを選びます。 入力したプロンプトに基づいて Copilot が行ったことを見てみましょう。

5. **[構成]** タブの **[名前]** と **[説明]** フィールドが、入力したプロンプトに基づいて設定されているはずです。 **[指示]** フィールドまで下スクロールします。 Copilot は、あなたが最初のプロンプトで指定した説明に基づいて、これらの指示を生成しました。 Copilot によって生成された詳細レベルの指示を確認します。

   > [!IMPORTANT]
   > エージェント ビルダーのプロセスが優れている点は、ユーザーによる基本的な自然言語の説明が、Copilot によって自動的に複雑な指示のセットに変換されることです。 このプロセスにより、ユーザーは詳細な指示のセットを自分で作成する必要がなくなります。

6. 指示を変更したい場合は、**[指示]** フィールドで直接手動で編集することも、代わりに指示を更新するよう Copilot に依頼することもできます。  
   あなたは、**[指示]** を確認した後、Copilot に他のいくつかの項目を指示セットに追加させることにしました。 これを行うには、画面左側のチャットで次のプロンプトを入力します。

   **次の項目を含めるよう指示を更新してください:**

   - **すべての回答にソースを引用してください。**
   - **憶測で答えないでください。情報が不足している場合やあいまいな場合は、その点を明示し、丁寧な代替回答を提示してください。たとえば、「割り当てられた人事ソースには確認済みの回答がありません。人事サポートに問い合わせるか、人事ポリシー ポータルを参照してください」などです。**
   - **機密情報やケース固有の話題 (報酬の詳細、医療データ、マネージャー限定のポリシーなど) については、プライバシーに配慮したメッセージで丁寧に断り、次のように導いてください: 「個人情報や機密情報にアクセスしたり、共有したりすることはできません。個別のサポートについては、人事サポートまでお問い合わせください。」**
   - **専門用語は避けてください。必要に応じて用語を簡単に定義してください。**

7. 指示を更新した後、Copilot の応答を確認します。 Copilot によって加えられた変更を確認するには、**[構成]** タブを選択し、**[指示]** フィールドまで下にスクロールします。 依頼した新しい指示が Copilot によって追加されたことを確認します。

8. 現在の指示は良さそうですが、改善の余地もありそうです。 どのように改良すればよいかよくわからないため、Copilot に意見を聞くことにします。  
   今度は、このエージェントを改良できる他の指示を提案するよう Copilot に依頼するプロンプトを入力します。

9. Copilot の推奨事項を確認します。 提案に満足したので、それらをすべてエージェントの指示に追加するよう Copilot に依頼します。

10. 指示を更新したという応答が Copilot からあったら、**[構成]** タブを選択し、**[指示]** をスクロールします。 Copilot が追加した新しい項目を確認します。

11. 指示に満足したら、エージェントのナレッジ ソースとスターター プロンプトを構成する準備は完了です。  
    **[構成]** タブで、**[ナレッジ]** セクションまで下にスクロールし、**[すべての Web サイトを検索する]** トグル スイッチが無効になっていることを確認します。 元のプロンプトで指定した説明に基づいてエージェントを作成する際には、提供したファイルのみを使用するように指示したため、Copilot により、このトグル スイッチは無効にされているはずです。 トグル スイッチが有効になっている場合は、ここで無効にします。

12. **[ナレッジ]** セクションの **[URL または名前を入力するか、ここにファイルをドロップしてください]** フィールドの横に表示される **[クラウドファイルを添付する]** アイコンを選択します。 表示される **アイテムを選択する** ウィンドウで **OneDrive** のマイファイルに移動し、OneDrive にコピーした 以下の6ファイルを選択します。

    - **Adatum Code of Conduct and Workplace Behavior.docx**
    - **Adatum Employee Benefits Policy.docx**
    - **Adatum Leave of Absence Policy.docx**
    - **Adatum Promotion and Career Development Policy.docx**
    - **Adatum Relocation Policy.docx**
    - **Adatum Remote Work and Flexible Schedule Policy.docx**

13. **[推奨プロンプト]** では、Copilot にプロンプトを生成してもらうか、手動で独自のプロンプトを作成することができます。 両方の方法を試してみましょう。  
    Copilot に推奨プロンプトを生成させるには、エージェント用の推奨プロンプトを 3 つ生成するよう Copilot に依頼します。 各プロンプトにどのようにタイトルとメッセージが作成されるかに注意してください。

14.  **[構成]** タブを選び、**[推奨プロンプト]** セクションまで下にスクロールします。 Copilot がエージェントに追加した 3 つのプロンプトが表示されます。  
    **[推奨プロンプトを追加する]** オプションを選択して、以下の 6 つのプロンプトを手動でエージェントに追加します。

        - **タイトル:** 転居ポリシー
            - **メッセージ:** 昇進に伴って転居する従業員に対する Adatum の転居ポリシーは何ですか。  
                
        - **タイトル:** 福利厚生の登録とプランの変更
            - **メッセージ:** 次回の福利厚生の登録期間はいつですか。また、医療プランまたは歯科プランを変更するにはどうすればよいですか。  
                
        - **タイトル:** 有給休暇 (PTO) と休暇
            - **メッセージ:** Adatum では PTO はどのように付与されますか。年末の繰り越しポリシーはどうなっていますか。PTO の申請方法を教えてください。  
                
        - **タイトル:** 昇格条件
            - **メッセージ:** Adatum で非管理職から管理職に昇進する際の一般的な昇進基条件は何ですか。  
                
        - **タイトル:** 給料日、控除、給与明細
            - **メッセージ:** 通常の給料日はいつですか。給与明細はどこで確認できますか。一般的な控除 (給付金、税金、退職金) はどのように処理されますか。  
                
        - **タイトル:** 育児休暇
            - **メッセージ:** Adatum の育児休暇ポリシーは何ですか。  

15. プロンプト候補をいくつかテストして、エージェントがデータをナレッジ ソースのドキュメントから正しく引き出していることを確認します。 画面上部にある **「試してみる」** をクリックしてタブを切り替え、表示されている推奨プロンプトのボックスをクリックして入力されたプロンプトを送信して、回答内容が指示通りに出力されているか確認します。

16. プロンプト候補の結果に満足したら、画面右上の **[作成]** ボタンを選択してエージェントを作成します。

17. エージェントが作成されると、エージェントが正常に作成されたことを示すダイアログ ボックスが表示されます。 このダイアログ ボックスでは、エージェントに移動したり、それを共有したりできます。 **[エージェントに移動する]** オプションを選択します。

> [!NOTE]
> この段階では、エージェントは非公開であり、アクセスできるのは自分だけです。 エージェントを複数のチーム メンバーが使う必要がある実際のシナリオでは、それらのユーザーと共有します。
